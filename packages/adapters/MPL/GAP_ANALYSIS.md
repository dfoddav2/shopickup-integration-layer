## Service Type Configurability

**Yes — service types are fully configurable.**

### Via canonical service levels

The canonical `Parcel.service` field accepts:
- `standard` → `A_175_UZL` (domestic standard)
- `express` → `A_121_CSG` (faster domestic parcel)
- `economy` → `A_175_UZL` (same as standard for MPL)
- `overnight` → `A_121_CSG` (fastest available domestic)

For international shipments:
- `standard` → `A_123_EUP` (Europe standard)
- `express` → `A_13_EMS` (Express Mail Service)

### Via exact carrier service code

Integrators can bypass the canonical mapping entirely by setting `parcel.carrierServiceCode` to any valid MPL basic service code:

```ts
parcel.carrierServiceCode = 'A_121_CSG'; // Exact MPL code
```

Valid codes:
- `A_175_UZL` — Belföld alapszolgáltatás (standard domestic)
- `A_177_MPC` — MPC (larger volume)
- `A_122_ECS` — Economy közép
- `A_121_CSG` — Csomag (faster domestic)
- `A_13_EMS` — Express Mail Service (international)
- `A_123_EUP` — Európa (international standard)
- `A_123_HAR` / `A_123_HAI` — Közlekedési + Ajánlott
- `A_125_HAR` / `A_125_HAI` — Csere-csomag
- `A_176_NET` / `A_176_NKP` — Nemzetközi

### Via extra services

Additional paid services can be specified explicitly via `options.mpl.extraServices`:

```ts
options.mpl.extraServices = ['K_IDA', 'K_FNK'];
```

These are merged with auto-derived extras:
- `K_ENY` — value insurance (auto-added when `declaredValue` or `insurance` present)
- `K_TER` — bulky handling (auto-added when `fragile` or `perishables` handling present)
- `K_UVT` — cash on delivery (auto-added when `cod` present)
